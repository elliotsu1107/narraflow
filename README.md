# 文脉 Narraflow

> 小说/故事章节的写作工作流技能。Narraflow = Narrative + Workflow。

---

## 版本历史

### v1.4（当前）
- **新增 Python 字数统计脚本**：`scripts/wc.py` 自动提取正文统计中文字符+英文单词，精确可重复
- **自动扩写校验循环**：字数不足时自动扩写→脚本校验循环，直到达标，无需用户介入确认
- **零中断确认流程**：字数校验全自动运行，去除行数估算，统一用脚本字数
- **继承 v1.3 特性**：分段写作逐段自动生成+初稿自动保存+字数自动输出

### v1.3
- **分段写作改为逐段自动生成**：按段落顺序连续输出，段落之间不停顿等待
- **初稿自动保存**：生成后自动保存文件并输出实际字数
- **行数估算**：段落行数 × 25 作为字数参考（v1.4 起被 Python 脚本取代）

### v1.2
- **阶段六改为"确认后同步"**：精修+去AI化后先提交用户确认，确认无误后再更新文档
- **字数统一入口**：新增【字数标准】区块，所有字数要求统一引用，修改一处即可全局生效
- **段落字数不固定**：章节结构中的字数分配由内容决定，不套固定模板
- **整体优化**：章节结构规范、风格原则、局部扩写规则、注意事项全面升级

### v1.1
- **分段写作**：按段落顺序逐段生成，每段写完确认字数量后再写下一段
- **局部扩写**：字数不足时只在该段落内扩充，禁止重新生成整章
- **章节结构规范**：6种段落功能类型 + 字数占比 + 写作层次规范
- **风格原则**：叙事密度、AI感防范、节奏控制、一致性四大类15条规则
- **创作红线**：补充记忆唤醒为第一条红线

### v1.0
- 初始版本：六阶段写作流程
- 记忆唤醒 → 章节简述 → 写初稿 → 审阅 → 精修+去AI化 → 文档同步

---

## 命名

**文脉** — 意为"行文的脉络"，强调写作逻辑的连贯性和一致性。

**Narraflow** — Narrative（叙事）+ Workflow（流程）的组合，表明这是一个专注于叙事流程管理的工作辅助工具。
