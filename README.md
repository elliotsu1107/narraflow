## 命名

**文脉** — 意为"行文的脉络"，强调写作逻辑的连贯性和一致性。

**Narraflow** — Narrative（叙事）+ Workflow（流程）的组合，表明这是一个专注于叙事流程管理的工作辅助工具。

---

## 架构说明

> ⚠️ **重点：文脉是通用工作流，与具体小说项目分离。**

文脉 Narraflow 本身只提供写作流程和字数校验工具，**不包含任何小说内容**。使用时需配合独立的小说项目目录。

```
Narraflow/                    ← 文脉 skill（通用工作流）
├── SKILL.md                          六阶段写作流程定义
├── scripts/wc.py                     Python 字数统计脚本
├── README.md                         本文件
└── .claude/                          Claude Code 配置

<novel-project-root>/                 ← 小说项目（内容数据）
├── <作品总览>/                        索引、进度、创作红线
├── <世界观>/                          世界设定、势力、法则
├── <地理>/                            区域文件
├── <人物>/                            人物卡片
├── <能力体系>/                        能力体系（名称因项目而异）
├── <剧情>/                            全书总纲、分卷大纲、分章细纲、伏笔管理
├── <章节原稿>/                        ← 写作产出的章节文件
├── <创作笔记>/
└── <素材库>/
```

### 如何开始

1. 创建你的小说项目目录（参考上述结构）
2. 在小说目录中启动 Claude Code
3. 调用 narraflow skill 开始写作

SKILL.md 中所有路径均为**相对路径**，相对于当前小说项目的根目录解析。其他小说项目只要遵循同样的目录约定，文脉即可直接使用，无需修改 skill。
